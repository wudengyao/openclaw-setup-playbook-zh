## 🏢 大厂龙虾对比（GitHub + 互联网）

> 说明：以下是“开源证据 + 官网/媒体证据”双源汇总；完整版见 [大厂龙虾对比研究](大厂龙虾对比/README.md)。

| 对象 | GitHub 信号 | 互联网信号 | 结论 |
| --- | --- | --- | --- |
| OpenClaw（基准） | `openclaw/openclaw`，约 313k⭐ / 59.7k forks | 官方文档与版本发布最完整 | 主线开源生态 |
| AutoClaw | 约 43 相关仓，头部约 76⭐ | 可检索到智谱相关入口页与媒体报道 | 偏“一键安装/本地执行”产品化包装 |
| KimiQClaw（Kimi Claw） | `KimiQClaw` 同名仓弱（接近 0） | `kimi.com` 资源页有“Kimi Claw”完整产品叙事 | 命名分裂，建议按“Kimi Claw 产品层”描述 |
| MaxClaw | 约 17 相关仓，头部约 144⭐ | `maxclaw.ai` 主打云托管、10秒部署、多渠道 | 偏托管平台路线，开源主仓不集中 |
| QClaw | 约 37 相关仓，WeChat 周边项目热度高（723⭐/464⭐） | `qclaw.qq.com`（腾讯）强调微信直连、本地执行 | IM 场景强，但企业需重点审计合规 |

### 对比结论（首页版）

1. OpenClaw 仍是最强“可审计开源主线”。  
2. AutoClaw / MaxClaw / QClaw 更偏“产品化封装+托管/入口能力”。  
3. KimiQClaw 作为关键词在 GitHub 不强，但 Kimi Claw 在互联网产品叙事强。  
4. 采用建议：生产优先看开源可审计性，产品能力按官网与媒体交叉验证。

## 🚀 最短路径（3 分钟）

1. 安装 OpenClaw（推荐安装脚本）
2. 运行向导：`openclaw onboard --install-daemon`
3. 打开面板：`openclaw dashboard`
4. 开始聊天并逐步接入 Telegram / Slack / WhatsApp 等渠道

完整步骤见 [01-安装与快速上手](docs/01-install-and-quickstart.md)。
