## 2. 10 个高价值实战场景

## 案例 1：PR 自动评审回传 Telegram

**场景**：代码提交后自动评审，并把结果推送到 Telegram。  
**价值**：缩短评审反馈链路，提高 merge 速度。  
**实现要点**：
- 配置 GitHub + Telegram 渠道
- 使用 OpenClaw 触发 PR diff 分析
- 输出建议与“可合并/需修复”结论

---

## 案例 2：本地酒窖/库存管理技能

**场景**：把本地 CSV 导入，自动生成库存技能和查询流程。  
**价值**：把“杂乱表格”升级成自然语言可查系统。  
**实现要点**：
- 约定 CSV 模板
- 解析 + 数据清洗
- 提供库存查询、统计、补货建议

---

## 案例 3：电商/超市自动下单（Browser 自动化）

**场景**：按固定购物清单自动选品、加购、预约配送。  
**价值**：把重复劳动自动化，节省日常时间。  
**实现要点**：
- 浏览器自动化流程脚本
- 风险步骤（支付）设为人工确认
- 执行结果回传 Telegram

---

## 案例 4：Slack 自动客服 + 告警转发

**场景**：监控企业 Slack 支持频道，自动响应常见问题并转发重点告警。  
**价值**：7x24 首响，减少人工值班压力。  
**实现要点**：
- 频道监听 + FAQ 策略
- 高优先级消息转发到 Telegram/短信
- 错误处理与人工兜底

---

## 案例 5：语音电话助手（Voice Bridge）

**场景**：通过电话与代理对话，做提醒、查询、任务操作。  
**价值**：解放双手，适合通勤和无屏场景。  
**实现要点**：
- 语音通道桥接
- TTS/STT 质量与延迟优化
- 敏感操作二次确认

---

## 案例 6：多源技术资讯日报

**场景**：聚合 RSS / X / GitHub / 网页检索，生成高质量日报。  
**价值**：信息去噪，提升决策效率。  
**实现要点**：
- 主题过滤 + 去重
- 质量打分 + 摘要模板
- 固定时段推送

---

## 案例 7：会议纪要与任务自动分发

**场景**：会议结束后自动产出纪要，并同步到 Jira/Linear/Todoist。  
**价值**：减少会后执行断层。  
**实现要点**：
- 结构化纪要模板
- Action Item 抽取
- 人员与截止时间自动映射

---

## 案例 8：家庭日程/家务协同助手

**场景**：聚合家庭日历、预约、待办、库存提醒。  
**价值**：降低家庭协作沟通成本。  
**实现要点**：
- 多账户日历汇总
- 每日晨报推送
- 事件冲突提醒

---

## 案例 9：知识库与语义检索（Second Brain）

**场景**：把聊天、网页、文档沉淀为可检索知识库。  
**价值**：长期积累，越用越聪明。  
**实现要点**：
- 文档落地与索引策略
- 语义召回 + 关键词混合检索
- 结果溯源链接

---

## 案例 10：本地 CRM 自动化（纯本地优先）

**场景**：从邮件与日历自动发现联系人，建立 CRM 关系网。  
**价值**：对外商务和客户跟进更系统。  
**实现要点**：
- 本地数据库（如 DuckDB）
- 联系人抽取与去重
- 跟进提醒与 pipeline 统计

## 3. 可复用落地模板（建议）

任何案例都可按下面 5 步复制：

1. **定义目标**：一句话描述成功标准（如“每天 8:30 输出技术简报”）
2. **定义输入源**：消息源/API/文件来源
3. **定义动作链**：抓取 → 清洗 → 推理 → 输出
4. **定义安全边界**：哪些动作必须人工确认
5. **定义验收指标**：准确率、耗时、人工节省量

---

## 4. 安全提醒（非常重要）

- 社区技能是“可用资源”，不是“默认可信”。
- 上线前至少做三件事：
  1) 看源码；2) 查权限；3) 用最小权限运行。
- 涉及金钱、删除、外发消息的动作，必须加人工确认步骤。
